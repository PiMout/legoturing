INTRODUCTION:

    A Turing machine is a finite state machine (FSM) that is built basically by three elements:
 
  • A tape of infinity length, divided in cells with one symbol (and only one) belonging to some alphabet. We will use a binary alphabet for the tape.
  • Over the tape there is a device able to move along it one cell each time. This device has a bolster able to read a symbol written in the tape, or to delete and print a new one.
  • Finally, it has a register able to store a state, that is defined by a symbol. The symbols that define the device state do not have to be the same as the ones could be read or written over the tape. We will use natural numbers for representing the several states of the machine.


    From a working point of view, the machine has a sequential behavior completely defined:

    • It starts from any state in the register and with the bolster in a tape position.
    • The bolster read the symbol in the cell under it.
    • With this input information and in function of the machine state, it access a table (program) and obtains an output with the symbol for printing over the tape (in the actual position), the new state of the machine and the next cell the bolster goes (left or right).
    • It goes to step 2.
    With this, we can define the finite state automata MT as:

 MT = < S, (SxM) U (stop), Q, f, g >

    being:

        • S finite set of symbols in the tape (0, 1)
        • M finite set of movements of the bolster (left = <, right = >)
        • Q finite state of internal states (q = 0, 1, 2, 3 ...)
        • f: S x Q -> Q transition function
        • g: S x Q -> (S x M) U (stop) output function

TIPYCAL NUMBERINGS IN A TURING MACHINE

  • UNARY NUMBERS (UN)
       We could think about a binary representation for the numbers, but the amount of operations for managing them with this format makes that the number of states for the machine raises very much.

        For simplifying the process, we can represent each number with a string with as 1 as the cardinal for each one (Unary Numbers). Thus:

                                        2 = 11
                                        5 = 11111

  •  EXPANDED BINARY NUMBERS (XN)
      •    But if we treat with big numbers, unary numbers dont seem a good representation, so we can introduce a new type of numbering based in binary representation, but with some modifies to allow Turing Machine easy understanding. This new kind of number representation is called Expanded Binary.

      Steps to follow to represent a number in expanded binary:
       

        1. Convert the number to binary.
        2. Codify the binary number according to the next codification table:

          3. Substitute a '0' for a '0'
          Substitute a '1' for a '10'
        3. And Use '110'(=,) to separate or end numbers (to avoid confusing the Turing Machine)


      Example:  Represent the number 105 in Expanded Binary:
       

        1. In binary:       1101001
        2. Codifying:      10100100010
        3. To end:          10100100010110

    Example 1: Unary+Unary

        Let's see a first simple example for understanding how works Turing Machine. One of the simple tasks that can be done with a Turing Machine is adding two unary numbers.

        First we must define a convention for representing such numbers over the tape. We could think about a binary representation of them, but the amount of operations for managing them with this format makes that the number of states for the machine raises very much.

        For simplifying the process, let's represent each number with a string with as 1 as the cardinal for each one. Thus:

                           2 = 11
                           5 = 11111

        Both numbers will be split by a zero:

     ... 0000110111110000 ...

        Where (...) means an infinite number of zeros.

        For getting the result from this representation, a solution could be convert the intermediate 0 in a 1 and delete a 1 in a extreme writing a zero instead of it.
     


     ... 000011111110000 ...


        In this way we have a string with a number of ones equal to the sum of both numbers.

        For implementing this, we suppose as initial conditions that the bolster is located to the left of  the first one. The register will be in the internal state A (for this example we will use letters instead of natural numbers for internal states).

        Therefore, the machine should move to the right until if finds a 1, change it by a 0, move to the right until it finds a 0 and change it by a 1.

        The next program will do this algorithm:
     
     
    READ SYMBOL
    READ SYMBOL
    INTERNAL STATE
    0
    1
    A
    0,A,>
    0,B,>
    A
    1,stop
    1,B,>

        If the machine reads a 0 (top row) and it is in A state then it does (0, A, >), i.e. it prints a 0, remain in A state and it moves to the right.

        If it is in A state and reads a 1, it prints a 0. changes to B state and goes to the right.

        It it is in B state and reads a 1, it prints a 1, remain in B state and moves the bolster to the right.

        It it is in B state and reads a 0. prints a 1 and stop

    Program:
     
    0 0,0,> 0,1,>
    1 1,stop 1,1,>
    F

    Example 2: Unary*2 (UN*2)

    This program duplicates an unary number

    Example: number 4

    ...00011110000000...
    Solution: 4*2=8
    ...00011111111000...


    Program:
     
    0 0,0,> 0,1,>
    1 1,2,< 1,1,>
    2 1,3,> 0,4,>
    3 0,stop 1,3,>
    4 1,5,< 1,4,>
    5 1,2,< 1,5,<
    F

    Example 3: Expanded Binary+1 (XN+1)

    Add one to an Expanded Binary Number.

    Example:  number 41 (binary: 101001)

    ...000100100010110000...  (41 and the separator 110)
    Solution: 41+1=42 (binary: 101010)
    ...000100100100110000...  (42 and the separater 110)
    Program:
     
    0 0,0,> 1,1,>
    1 0,0,> 1,2,>
    2 0,3,< 1,2,>
    3 1,stop 0,4,<
    4 1,5,< 1,4,<
    5 0,6,> 1,2,>
    6 0,stop 1,7,>
    7 1,3,> 0,7,>
    F

    Example 4: Expanded Binary*2 (XN*2)

    Duplicate a Binary Expanded  number.

    Example: number 41 (binary: 101001)

    ...000100100010110000...(41 and the separator 110)
    Solution: 41*2=82 (binary: 1010010)
    ...000100100010011000...(82 and the separator 110)
    Program:
     
    0 0,0,> 0,1,>
    1 1,0,> 0,2,>
    2 1,3,> 1,stop
    3 1,stop 1,stop
    F
